# Foundation Models for Hydro-Climate Forecasting

Official code for **"Foundation Models for Hydro-Climate Forecasting: Zero-Shot and Few-Shot Transfer Across Global Basins"** — submitted to *Environmental Modelling & Software*.

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
[![Python 3.9+](https://img.shields.io/badge/python-3.9+-blue.svg)](https://www.python.org/downloads/)

---

## Overview

We benchmark three time-series foundation models — **TimesFM**, **Chronos**, and **PatchTST** — against a persistence baseline for zero-shot daily streamflow forecasting across six global CAMELS datasets (6,454 basins, six continents). We also evaluate Chronos LoRA few-shot fine-tuning on three datasets.

**Key finding**: Foundation models are competitive with — but do not universally outperform — a naive persistence baseline. Performance depends strongly on regional autocorrelation structure. TimesFM leads in 5 of 6 datasets; PatchTST fails in all datasets (median NSE ≈ 0).

## Results

### Zero-Shot Performance (Median NSE, test period 2015–2019)

| Dataset | Basins | Chronos | TimesFM | PatchTST | Persistence |
|---------|--------|---------|---------|----------|-------------|
| CAMELS-US | 673 | 0.547 | 0.507 | 0.003 | 0.465 |
| CAMELS-BR | 2,670 | 0.772 | 0.791 | 0.022 | **0.805** |
| CAMELS-CL | 430 | 0.795 | 0.806 | 0.020 | **0.818** |
| CAMELS-AUS | 560 | 0.310 | **0.402** | 0.003 | 0.334 |
| LamaH-CE | 880 | 0.579 | **0.690** | 0.005 | 0.690 |
| CAMELS-IND | 241 | 0.369 | **0.501** | 0.003 | — |

Bold = best model per dataset. CAMELS-IND persistence baseline not computed (no overlap in temporal coverage for shift-based estimate).

### Few-Shot Fine-Tuning: Chronos LoRA (Median NSE)

| Dataset | Zero-shot | f=0.01 | f=0.05 | f=0.10 | f=0.25 |
|---------|-----------|--------|--------|--------|--------|
| CAMELS-BR | 0.772 | 0.745 | 0.748 | 0.741 | 0.770 |
| CAMELS-AUS | 0.310 | **0.442** | 0.429 | 0.429 | 0.348 |
| CAMELS-IND | 0.369 | 0.348 | 0.328 | 0.345 | 0.336 |

Few-shot helps substantially in Australia (+0.132 at f=0.01), provides no benefit in Brazil (high autocorrelation ceiling), and slightly hurts in India (distribution shift).

## Setup

### 1. Install Dependencies

```bash
git clone https://github.com/zechariahyue/hydro-foundation-model.git
cd hydro-foundation-model
pip install -r requirements.txt
```

### 2. Configure Paths

Set environment variables pointing to your data, or edit `configs/experiment_config.yaml`:

```bash
export CAMELS_RAW_DIR="/path/to/raw/camels"        # raw downloaded datasets
export CAMELS_DATA_DIR="/path/to/processed/camels" # preprocessed parquet files
export RESULTS_DIR="results"                        # output CSVs
export FIGURES_DIR="figures"                        # output figures
export CKPT_DIR="checkpoints"                       # model checkpoints
```

### 3. Download Datasets

| Dataset | Basins | Source |
|---------|--------|--------|
| CAMELS-US | 671 | https://ral.ucar.edu/solutions/products/camels |
| CAMELS-BR | 2,670 | https://zenodo.org/records/3709337 |
| CAMELS-CL | 516 | https://zenodo.org/records/3955968 |
| CAMELS-AUS | 222 | https://zenodo.org/records/3705993 |
| LamaH-CE | 859 | https://zenodo.org/records/4525244 |
| CAMELS-IND | 472 | https://zenodo.org/records/10019074 |

Basin counts after filtering: only basins with ≥365 days of discharge observations in the test period are retained (final counts in table above).

## Reproducing Paper Results

### Step 1: Preprocess Data

```bash
python data/preprocess_camels.py        # CAMELS-US/BR/CL/AUS/LamaH-CE
python data/preprocess_camels_ind.py    # CAMELS-IND
```

### Step 2: Train LSTM Baseline (optional)

```bash
python models/train_baselines.py --model lstm
```

### Step 3: Zero-Shot Evaluation

```bash
# TimesFM
python models/run_foundation_models.py --model timesfm --mode zero_shot

# Chronos
python models/run_foundation_models.py --model chronos --mode zero_shot

# PatchTST
python models/run_foundation_models.py --model patchtst --mode zero_shot
```

### Step 4: Few-Shot LoRA Fine-Tuning (Chronos)

```bash
for FRAC in 0.01 0.05 0.10 0.25; do
  for DS in CAMELS-BR CAMELS-AUS CAMELS-IND; do
    python models/run_few_shot_lora.py --model chronos --fraction $FRAC --dataset $DS
  done
done
```

### Step 5: Compute Statistics and Figures

```bash
python evaluation/compute_bootstrap_stats.py    # CIs + Wilcoxon tests
python evaluation/generate_figures.py           # paper figures
python analysis/physical_diagnostics.py         # FDC-KGE, PBIAS
python analysis/sensitivity_analysis.py         # context-length sensitivity
```

## Model Configuration

Key settings in `configs/experiment_config.yaml`:

| Parameter | Value |
|-----------|-------|
| Context length | 512 days |
| Forecast horizon | 1 day |
| Test period | 2015-01-01 to 2019-12-31 |
| TimesFM | `google/timesfm-2.0-500m-pytorch` |
| Chronos | `amazon/chronos-t5-large` |
| PatchTST | `ibm/patchtst-etth1-forecast` |
| LoRA rank | 8, alpha 16 |
| LoRA target modules | q_proj, v_proj |
| Few-shot fractions | 0.01, 0.05, 0.10, 0.25 |
